## نظرة عامة

يهدف التطبيق إلى توفير تجربة تعليمية شاملة ومحفزة تغطي كل مهارات اللغة الأربع:

| المهارة | كيف يُعلّمها التطبيق |
|---|---|
| **الاستماع (Listening)** | جمل صوتية مُركَّبة + بطاقات مطابقة تفاعلية |
| **الكلام (Speaking)** | تحقق صوتي بالميكروفون عبر Web Speech Recognition |
| **القراءة (Reading)** | نصوص قصيرة + أسئلة فهم متعددة الاختيار بشرح عربي |
| **الكتابة (Writing)** | بناء جمل من كلمات مبعثرة + ملء الفراغات النحوية |

**إجمالي المحتوى:** 1000 كلمة إنجليزية، 40 اختباراً، 40 جلسة إتقان مفردات

### بنيانات بيانات الدرس

كل درس (`lesson`) يحتوي على:

```javascript
{
  id: Number,           // رقم الدرس التسلسلي (1-40)
  unitId: Number,       // رقم الوحدة الأم (1-10)
  title: String,        // عنوان الدرس بالإنجليزية
  description: String,  // وصف مختصر للدرس
  explanation: {
    intro: String,      // مقدمة بالعامية المصرية
    sections: [         // أقسام الشرح النظري
      { title, content }
    ]
  },
  vocabulary: [         // مفردات الدرس متغيرة العدد
    {
      id: Number,
      word: String,
      type: String,           // noun, verb, adjective, phrase, interjection
      translation: String,    // الترجمة المصرية
      example: String,        // جملة مثال إنجليزية
      exampleTranslation: String
    }
  ],
  quiz: [               // 5 أسئلة اختبار
    {
      type: String,     // multiple-choice, fill-blank, translate, match
      question: String,
      options: [],
      answer: String,
      explanation: String  // شرح الإجابة بالعامية
    }
  ],
  exercises: {          // تمارين التدريب الرباعي
    listening: [],
    speaking: [],
    reading: [],
    writing: []
  }
}
```

## المميزات التفصيلية

### 1. لوحة التحكم (Dashboard)

**حلقة التقدم SVG:** رسم ديناميكي لدائرة التقدم يُحسب بدقة من نسبة الدروس المكتملة إلى الـ 40 درساً.

**بطاقات الإحصائيات الأربع:**
- نسبة إتمام الدورة (%)
- إجمالي XP المكتسبة
- عدد الدروس المكتملة
- أسئلة الاختبار التي أُجيب عليها

**لوحة الأداء بالمهارة:** شريط تقدم لكل مهارة (Listening, Speaking, Reading, Writing) محسوب من نتائج جلسات التدريب.

**الدرس التالي المقترح:** زر مباشر يوجه لأول درس غير مكتمل في المسار.

---

### 2. خريطة التعلم (Roadmap)

**نظام القفل التسلسلي:**
يُفتح الدرس `N+1` فقط عند اجتياز **شرطين معاً** في الدرس `N`:
1. اجتياز اختبار الدرس بنسبة **≥ 60%** (3 من 5 أسئلة)
2. إتمام تلاوة مفردات الدرس بنجاح (النطق + التهجئة)

**ثلاث حالات للدرس:**
- `locked` — لم تُستكمل شروط الفتح، يظهر بشفافية 75%
- `unlocked` — متاح للدراسة، حد Indigo بنفسجي
- `completed` — مكتمل، حد أخضر Emerald

**عرض الوحدات:** كل وحدة قابلة للطي (Collapsible) مع عداد `X / 4 Completed` لعرض التقدم الجزئي.

---

### 3. عارض الدروس (Lesson Viewer)

**تبويبات ثلاث داخل كل درس:**

| التبويب | المحتوى |
|---|---|
| `Learn` | الشرح النظري بالعامية + فقاعات المعلم التفاعلية |
| `Vocabulary` | بطاقات 10 كلمات مع زر استماع TTS وترجمة مصرية |
| `Quiz` | 5 أسئلة تفاعلية مع شرح فوري لكل إجابة |

**الانتقال السياقي:** عند إتمام الاختبار بنجاح يُوجَّه الطالب مباشرة لمركز المفردات للدرس نفسه.

---

### 4. مركز المفردات والتحقق المزدوج (Vocab Center)

يمر الطالب بمرحلتين **لكل كلمة من الكلمات العشر:**

**المرحلة الأولى — النطق الصوتي:**
1. يُنطق الكلمة بصوت آلي (TTS) تلقائياً
2. يضغط الطالب زر الميكروفون وينطق الكلمة
3. يقارن النظام المدخل الصوتي بالكلمة المستهدفة (مطابقة غير حساسة للحالة، تجاهل علامات الترقيم)
4. في حال فشل التعرف أو عدم دعم المتصفح، يمكن تجاوزها والانتقال للتهجئة مباشرة

**المرحلة الثانية — التهجئة الكتابية:**
1. يكتب الطالب الكلمة من الذاكرة
2. **محرك تحليل الأخطاء الذكي** يفحص كل حرف:

```
مثال 1: الكلمة "welcome"، كتب الطالب "welzome"
النظام يقول: "الحرف الخامس خطأ (كتبت z والمفروض c)"

مثال 2: الكلمة "welcome"، كتب الطالب "welcom"
النظام يقول: "الحرف السابع ناقص"

مثال 3: الكلمة "welcome"، كتب الطالب "welcomee"
النظام يقول: "الكلمة أطول من اللازم (حرف زيادة في الآخر)"
```

---

### 5. مركز المراجعة اليومية (Review Center)

**لوحة التخصيص:**

| الإعداد | النطاق | الخطوة |
|---|---|---|
| عدد كلمات المراجعة | 1 – 20 | 1 |
| تمارين الاستماع | 0 – 5 | 1 |
| تمارين الكلام | 0 – 5 | 1 |
| تمارين القراءة | 0 – 5 | 1 |
| تمارين الكتابة | 0 – 5 | 1 |

**الجلسة الافتراضية الديناميكية:**

```
عدد الأسئلة الافتراضية = 8 + (عدد الدروس المكتملة × 2)
الحد الأقصى = 30 سؤالاً
```

**HUD ما قبل الجلسة:** يعرض إجمالي الأسئلة ومكافأة XP المتوقعة (`+10 XP` لكل سؤال) قبل النقر على Start.

**آلية اختيار الكلمات:** تُختار عشوائياً من مجموع المفردات في الدروس المكتملة مع إعطاء الأولوية للكلمات الأقل مراجعة.

---

### 6. محرك التدريب الرباعي (Practice Engine)

#### الاستماع (Listening)
- يُنطق جملة إنجليزية صوتياً (TTS)
- يظهر للطالب 4 بطاقات: يختار الكلمة التي سمعها في الجملة
- شرح فوري بالعامية عند الإجابة

#### الكلام (Speaking)
- تحدي النطق: يظهر الكلمة ويطلب نطقها صوتياً
- مطابقة باستخدام `SpeechRecognition API`
- بديل كتابي في المتصفحات غير الداعمة

#### القراءة (Reading)
- نص قصير مناسب للمستوى (3-5 جمل)
- سؤال فهم واحد متعدد الاختيار
- شرح الإجابة يُفسَّر بالعامية المصرية

#### الكتابة (Writing)
- **Scrambled Sentence:** كلمات مبعثرة تحتاج ترتيباً صحيحاً (Drag & Drop بديل: بطاقات قابلة للنقر)
- **Fill-in-the-blank:** جملة ناقصة مع اختيار الكلمة الصحيحة نحوياً

---

### 7. القاموس التفاعلي (Dictionary)

- فهرسة كاملة لـ **1000 كلمة** فريدة من المنهج
- بحث فوري (Live Search) بالإنجليزية أو العربية
- فلترة حسب الوحدة (Unit 1-10) أو الدرس المحدد (1-40)
- كل بطاقة كلمة تعرض: الكلمة، النوع، الترجمة، جملة المثال، ترجمة المثال، وزر الاستماع

---

### 8. الاختبار النهائي والشهادة (Final Exam & Certificate)

**شروط الوصول:** إتمام جميع الدروس الـ 40 أو اجتياز حد أدنى من الوحدات.

**الاختبار:**
- 20 سؤالاً من بنك شامل يغطي كل الوحدات
- يُوزَّع السؤال عشوائياً من كل وحدة لضمان التوازن
- عرض فوري لحالة كل إجابة مع الشرح

**الشهادة:**
- تُفعَّل عند اجتياز **80% (≥16 من 20)**
- يُدخل الطالب اسمه الكامل ليُطبع على الشهادة
- تصميم شهادة احترافي بإطار ذهبي مزدوج وزوايا مزينة
- دعم `@media print` للطباعة المباشرة أو الحفظ كـ PDF

## المعمارية التقنية

### نمط SPA Router (Hash-based Routing)

```
URL Hash  ──►  ROUTES Map  ──►  Section DOM ID  ──►  Component Function
────────────────────────────────────────────────────────────────────────
#dashboard        →  dashboard-section       →  renderDashboard()
#roadmap          →  roadmap-section         →  renderRoadmap()
#lesson-viewer    →  lesson-viewer-section   →  renderLesson(id, tab)
#vocab-center     →  vocab-center-section    →  initVocabCenter(params)
#dictionary       →  dictionary-section      →  renderDictionary()
#final-test       →  final-test-section      →  initFinalTest()
#review-center    →  review-center-section   →  initReviewCenter()
```

**الانتقال بالمعاملات:**
```
#lesson-viewer?id=5&tab=quiz    ──►  فتح الدرس 5 على تبويب الاختبار
#vocab-center?lesson=3          ──►  مركز مفردات الدرس 3 مباشرة
```

### دورة تحديث الواجهة

```
حدث مستخدم (إجابة/إتمام)
      │
      ▼
saveProgress()  ──►  localStorage
      │
      ▼
window.dispatchEvent("academyProgressUpdated")
      │
      ▼
updateProfileUI()  ──►  تحديث XP Bar + الرتبة + الأفاتار
```

### تدفق إتمام الدرس

```
اجتياز الاختبار (≥60%)   ──┐
                             ├──►  checkAndCompleteLesson(id)
إتمام التلاوة (10/10)    ──┘         │
                                      ▼
                              كلا الشرطين؟
                                   YES  ──►  completedLessons.push(id)
                                             +150 XP Bonus
                                   NO   ──►  انتظار استيفاء الشرط الآخر
```

## نظام الصوت والمؤثرات

### Web Speech API — SpeechSynthesis (TTS)
- لغة إنجليزية (en-US / en-GB)
- سرعة النطق قابلة للتعديل: من `0.5x` إلى `1.5x` عبر شريط تحكم في الشريط الجانبي
- يُستخدم في: نطق المفردات، جمل الاستماع، قراءة النصوص

### Web Speech API — SpeechRecognition
- `continuous: false` / `interimResults: true`
- مطابقة الناتج مع الكلمة المستهدفة بعد تجريد علامات الترقيم والحالة
- توفر fallback لـ Skip إلى مرحلة الكتابة عند عدم الدعم

### Web Audio API — Audio Effects Engine

محرك صوتي مُركَّب داخلياً بدون ملفات صوتية خارجية:

| الحدث | التوليف الصوتي |
|---|---|
| إجابة صحيحة | نغمة مزدوجة C5 → E5 (chime لطيف) |
| إجابة خاطئة | نغمتان G3 → F3 بموجة مثلثية (تحذير ناعم) |
| إتمام / احتفال | أربيجيو C5 → E5 → G5 → C6 (fanfare) |
| نقر / تفاعل | نقرة قصيرة 600Hz |

**التحكم:** زر SFX On/Off يحفظ الحالة في LocalStorage.

## نظام التصميم

### لوحة الألوان (HSL Tokens)

```css
/* Dark Mode */
--bg-primary:    hsl(220, 24%, 7%)   /* خلفية التطبيق الرئيسية */
--bg-secondary:  hsl(220, 24%, 11%)  /* الكروت والألواح */
--bg-tertiary:   hsl(220, 18%, 16%)  /* حقول الإدخال والتفاصيل */
--primary:       hsl(250, 84%, 54%)  /* اللون الأساسي (Indigo) */
--accent:        hsl(38, 92%, 58%)   /* الذهبي (XP والإنجازات فقط) */
--success:       hsl(150, 76%, 50%)  /* الأخضر (Correct / Completed) */
--error:         hsl(350, 89%, 60%)  /* الأحمر (Incorrect / Error) */
```

### نظام الـ 3D الاحترافي

يعتمد على ثلاث طبقات:

**1. Surface Gradient** — سطح الكارت يتدرج من أفتح (أعلى) لأغمق (أسفل) محاكياً الإضاءة الطبيعية:
```css
background: linear-gradient(180deg, var(--bg-secondary) 0%, var(--bg-tertiary) 100%);
```

**2. Multi-layer box-shadow** — طبقات ظل تُجسِّم العمق:
```css
box-shadow:
  inset 0 1px 0 rgba(255,255,255,0.08),  /* highlight علوي */
  0 2px 4px rgba(0,0,0,0.40),           /* ظل قريب */
  0 6px 12px rgba(0,0,0,0.35),          /* ظل متوسط */
  0 12px 24px rgba(0,0,0,0.25),         /* ظل بعيد */
  0 20px 40px rgba(0,0,0,0.15);         /* contact shadow */
```

**3. Glossy Shimmer Line** (Pseudo-element) — خط ضوئي على الحافة العليا:
```css
::before {
  background: linear-gradient(90deg, transparent, rgba(255,255,255,0.15) 40%, transparent);
  height: 1px;
}
```

### Utility Classes (Tailwind-like)

فئات عرض جاهزة من `w-0` إلى `w-100` تُستخدم للـ Progress Bars:
```html
<div class="xp-fill w-65"></div>  <!-- 65% عرض -->
```

## تشغيل المشروع

المشروع تطبيق ثابت (Static) لا يحتاج بناءً أو خادماً للقاعدة البيانات.

### الطريقة الأولى — فتح مباشر
```
انقر مرتين على index.html
```
> بعض المتصفحات تقيد أذونات الميكروفون عبر `file://`. استخدم الطريقة الثانية للتجربة الكاملة مع ميزات الكلام.

### الطريقة الثانية — خادم محلي (موصى به)

**Node.js:**
```bash
npx serve .
# يعمل على http://localhost:3000
```

**Python 3:**
```bash
python -m http.server 3000
# يعمل على http://localhost:3000
```

**VS Code:** تثبيت امتداد **Live Server** ثم النقر على `Go Live`.

افتح `http://localhost:3000` في **Google Chrome** أو **Microsoft Edge**.
